## 測試 4：天氣功能 ☁️

### 前置條件：
```
需要設定 OpenWeather API Key

1. 前往 https://openweathermap.org/api
2. 註冊免費帳號
3. 取得 API Key
4. 編輯 .env：
   OPENWEATHER_API_KEY=你的API_Key
5. 重新啟動伺服器
```

### 測試步驟：
```
1. 新增至少 1 個路線點
2. 點擊「查看天氣」按鈕
3. 等待載入
```

### 預期結果：
- ✅ 顯示天氣資訊
- ✅ 顯示溫度、天氣描述
- ✅ 顯示風速、濕度
- ✅ 顯示天氣圖示

### 如果失敗：
```
檢查項目：
□ API Key 是否正確
□ 是否重新啟動伺服器
□ 檢查 Console 錯誤訊息
□ 確認未超過免費額度
```

## 問題排除

### 問題 1：機車還是走高速公路
```
解決方法：
1. 清除瀏覽器快取（Cmd+Shift+R）
2. 重新啟動伺服器
3. 刪除所有路線點重新規劃
4. 檢查 Console 是否有錯誤
```

### 問題 2：搜尋後沒有彈出視窗
```
解決方法：
1. 檢查瀏覽器是否阻擋彈出視窗
2. 查看 Console 錯誤訊息
3. 確認 Places API 已啟用
4. 重新整理頁面
```

### 問題 3：每日行程按鈕沒出現
```
解決方法：
1. 確認已新增至少 2 個路線點
2. 重新整理頁面
3. 檢查 Console 錯誤
4. 清除快取重試
```

### 問題 4：附近搜尋沒反應
```
解決方法：
1. 先點擊搜尋類型按鈕（🏨、⛽ 等）
2. 確認按鈕變成藍色（active 狀態）
3. 再點擊路線點旁的 🔍
4. 檢查 Console 錯誤
```

### 問題 5：天氣無法顯示
```
解決方法：
1. 確認 .env 有 OPENWEATHER_API_KEY
2. 確認 API Key 正確
3. 重新啟動伺服器
4. 檢查是否超過免費額度
5. 查看 Console 錯誤訊息
```
